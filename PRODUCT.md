# Product

## Register

**product design** — the interface serves a task, not a brand impression. Users come to make carousels, not browse the tool. Every pixel either accelerates the workflow or is removed.

## Users

Content creators, social media managers, and marketers who need to produce polished LinkedIn and social media carousel posts quickly. They are visually literate, care about design quality, and will compare the output to Canva, Figma, and Pitch. They work in focused sessions: pick a theme, write content across slides, export, and post.

## Product Purpose

Carouselify lets users create professional carousel posts in minutes, not hours. The core value is eliminating the design bottleneck: pick a template, fill in content, export as PNG or PDF. Success is a user going from idea to posted carousel in under 10 minutes with output that looks like a designer made it.

## Brand Personality

Confident, precise, effortless. The tool should feel like a well-built instrument: every interaction is deliberate, nothing is decorative, the output speaks for itself. Not playful. Not corporate. Professional craft.

## Anti-references

- Canva's cluttered, template-browsing, everything-at-once interface
- Generic SaaS admin dashboards with identical card grids
- AI-generated landing pages with gradient text, glassmorphism, and cream backgrounds
- Tools that prioritize features over workflow (too many options, no clear path)

## Design Principles

1. **Output is the product.** The slide canvas is the hero. The editor chrome should feel like a confident instrument around it, not compete with it.

2. **Earned familiarity.** Standard patterns, consistent vocabulary, zero learning curve for anyone who has used a design tool. The tool disappears into the task.

3. **Speed as feature.** Every interaction should feel instant. No loading screens between actions, no confirmation dialogs for low-risk operations, no friction before export.

4. **Confident restraint.** One accent color used with purpose. Clean surfaces. No decorative elements that do not serve the workflow.

## Key Capabilities

- **5 slide types** (cover, content-b1, content-b2, list, cta) on a 1080x1080 canvas with 72px margins, 8px accent progress bar
- **10 color schemes** plus a custom scheme builder (saved per account)
- **4 font pairings** by default; **56 curated Google Fonts** for premium subscribers (any display + body combination)
- **Customizable logo** with 5 blob shapes, 4 positions, letter input, and premium image upload (PNG, JPEG, WebP, GIF, 2MB limit)
- **AI generation** (gpt-4o-mini): premium users get 50 credits per month; registered users get 5; guests get 1 free use
- **Export** as individual PNG files or multi-page PDF via html-to-image
- **Showcase gallery** at `/showcase` with community submissions and featured examples
- **Share** via permanent public link (one click, copies URL to clipboard); publishing to the showcase gallery is separate

## Accessibility & Inclusion

WCAG AA: 4.5:1 contrast for text, 3:1 for large text. Keyboard navigation for primary actions. Focus indicators on all interactive elements. Reduced motion support (`prefers-reduced-motion`). The tool should work for users with motor impairments (keyboard-only) and low vision (contrast).
